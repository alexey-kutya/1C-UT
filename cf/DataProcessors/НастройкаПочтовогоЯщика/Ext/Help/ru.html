Обработка "Загрузка почтовых сообщений"

 

Обработка предназначена для загрузки и формирования почтовых сообщений. Возможна работа непосредственно с POP3/SMTP серверами, либо с MS Outlook через COM-соединение, а также IBM Lotus.

В случае работы с POP3/SMTP необходимо задать параметры соединения:

  • имя сервера, а также порт через двоеточие, если используется нестандартный;
  • логин пользователя;
  • пароль пользователя;
  • способ аутентификации.

В случае, если нет необходимости получения почты, тогда достаточно не задавать имя сервера POP3, если нет необходимости в отправке почте, тогда достаточно не задавать имя сервера SMTP.

 

Есть возможность отправки/получения почты через приложение MS Outlook, в этом случае достаточно поставить признак "Использовать MS Outlook". Если этот признак установлен, тогда обработка будет пытаться установить COM-соединение с MS Outlook и отправлять или получать почту через это приложение. Требование успешной работы через внешнее соединение - установленное приложение MS Outlook на сервере приложений.

 

Есть возможность отправки/получения почты через приложение IBM Lotus, в этом случае достаточно поставить признак "Использовать IBM Lotus Notes", а также задать пароль пользователя, от чьего имени планируется отправка почтовых сообщений. Если этот признак установлен, тогда обработка будет пытаться установить COM-соединение с IBM Lotus Notes и отправлять или получать почту через это приложение. Требование успешной работы через внешнее соединение - установленное приложение IBM Lotus Notes на сервере приложений.

 

Обработка может работать как интерфейсно - в этом случае ее необходимо открыть через меню "Операции - Обработки ...", либо как задание, заданное в справочнике Задания. Во втором случае обработка будет запускаться в фоне на сервере приложений из регламентного задания по заданному расписанию.

 

Оправка писем:

Источником отправляемых сообщений является регистр сведений "Сообщения и уведомления" - обрабатываются все неотправленные сообщения - все записи с неустановленным признаком "Отправлено". В случае необходимости повторной отправки можно вручную сбросить признак отправки. Посмотреть регистр сведений можно непосредственно из формы документов по соответствующей кнопке, либо через меню "Операции - Регистры сведений ..."

 

Получение писем:

При разборе входящей почты по адресу отправителя выполняется поиск контактной информации с типом "E-Mail" по справочнику Физических лиц, Пользователей, Сотрудников. Если найдено соответствие, то будет создан новый документ Обращение, где в Инициатором будет выставлен найденный пользователь. Все обработанные письма, по которым найдены контакты в Итилиуме удаляются с сервера.

Если в письме будут заданы получатели копии письма, эти лица будет помещены в список оповещаемых лиц в новое Обращение.

Прикрепленные файлы к письму автоматически будут прикреплены к новому Обращению.

Дополнительно имеется возможность при получении писем не создавать новое Обращение, а по номеру, заданному в конце темы письма по шаблону "Инц.#0000000000" найти в базе Обращение и добавить содержимое письма в общение по Обращению.
